## Integration Pattern

Each component follows this pattern:

```typescript
// 1. Schema (sanity/schemas/blocks/aceternity/[component].ts)
export default defineType({
  name: "aceternity.[component-name]",  // ⚠️ Namespace required
  title: "Component Name",
  type: "object",
  fields: [
    { name: "items", type: "array", /* ... */ },  // ⚠️ Match component prop
    { name: "colorVariant", type: "color-variant" },
    { name: "padding", type: "section-padding" },
  ],
});

// 2. Block Wrapper (components/blocks/aceternity/[component]-block.tsx)
"use client";  // ⚠️ Required

import { Component } from "@/components/ui/[component]";
import SectionContainer from "@/components/ui/section-container";

export default function ComponentBlock({ items, colorVariant, padding }) {
  if (!items || items.length === 0) return null;

  return (
    <SectionContainer color={colorVariant as any} padding={padding as any}>
      <Component items={items} />
    </SectionContainer>
  );
}

// 3. UI Component (components/ui/[component].tsx)
// Original Aceternity component (may be "use client")
```

## Common Properties

All schemas include:

**Content Group:**
- `title` (optional) - Block heading
- `subtitle` (optional) - Block subheading
- Component-specific fields (`items`, `data`, etc.)

**Style Group:**
- `colorVariant` (color-variant) - Background color
- `padding` (section-padding) - Section spacing

**Data Requirements:**
- All array items need `_key` field
- All objects need `_type` marker
- Field names must match component props exactly

## Usage Notes

**Required for All Components:**
1. ✅ `"use client"` directive in block wrapper
2. ✅ `SectionContainer` wrapper
3. ✅ Null/empty checks
4. ✅ `aceternity.*` namespace
5. ✅ Proper TypeScript types

**Icon Handling:**
Some components (Bento Grid, Floating Dock) use hardcoded icon mapping:
```typescript
const iconMap: Record<string, React.ReactNode> = {
  workflow: <Workflow />,
  // ... more icons
};
```

**Schema Field Alignment:**
Critical: Schema field names must match component props:
- Timeline: `items` (not `data`)
- Bento Grid: `items` (not `gridItems`)
- Hero Parallax: `products` (component-specific)
